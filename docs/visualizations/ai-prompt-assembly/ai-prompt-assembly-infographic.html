How an AI Prompt Comes Together

Creating a full story with AI in CYOA Maker — from UI inputs to the final JSON sent to the AI, and back.

User input App data & settings Claude (text) OpenAI (images) Repeats in a loop
① What gets collected

From the user · Web UI

premisegenre toneaudience # scenes# endings scene lengthimage style image moodtheme* publish?
*Theme = colours + fonts only. It is stored but never sent to the AI.

From the app · data & settings

premises.json (seed) audiences.json play_fonts.json guardrails (settings) AI model (settings) prompt templates
Pulled in server-side as the prompt is assembled — the user never sees these.
▼
0

Resolve the inputs api_create_story_ai.php

Turn every “Any” into a concrete random value; fill a blank premise from a premises.json seed; pick one image style for the whole story. Save a draft story and queue a “story” job.
▼
1

Build the PLAN prompt → call Claude

Several sources merge into one template, then go out as a single JSON request.
story_plan_system.txt+ audience+ fonts+ guardrails =  system
premise · genre · tone · #scenes · #endings =  user message
POST api.anthropic.com/v1/messages { "model": settings, "max_tokens": 4000, "system": "…template + guardrails…", "messages": [ { role:"user", content:"premise/genre/tone…" } ] }
🤖 Claude → returns Plan JSON · title, scenes, branching choices, theme
Response is JSON-validated (1 retry), guardrail-checked (red_flag), sanitized.
▼
↻ LOOP — once per scene
2

Write each scene → call Claude × number of scenes

A second template (story_scene_writer_system.txt + tone + scene type + audience + guardrails) merges with the full story map and the one scene to write.
🤖 Claude→ returns Scene JSON · title, description, hint, image_prompt
max_tokens 1200 · temperature 0.8 · same request shape as stage 1.
▼
3

Save & queue images ai_apply.php

Write scene + choice rows (remap temp IDs). If images were included, queue one image job for the cover + one per scene that produced an image_prompt.
▼
↻ LOOP — once per image
4

Generate each image → call OpenAI × number of images

An image template merges with the scene's prompt + chosen style + mood (no theme).
image_system.txt+ image_prompt+ style+ mood
🎨 OpenAI→ returns image → saved to images/stories/{id}/
▼
5

Finalize

When the plan, every scene, and every image succeed — and the publish box was ticked — the story auto-publishes. Any failure leaves it a draft.
The "why is it so many calls?" math
1 plan+ S scene calls+ I image calls = total AI calls
One request fans out into many. The plan is a single call; scenes and images are loops. A 12-scene story with images can be ~25+ AI calls — which is why generation runs as a background job, not on the page.
