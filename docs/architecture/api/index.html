CYOA Maker — Internal API

Version: 1.0.0

Internal AJAX / JSON endpoints used by the CYOA Maker web app. These are not a public REST API: each endpoint is a single PHP file that dispatches on an action query/body parameter, authentication is the PHP session cookie, and request bodies are application/x-www-form-urlencoded (browser form posts), not JSON.

Because of the action-dispatch design, one path serves several logical operations — the per-action request fields and response shapes are documented via descriptions and response examples. Mutations use POST; reads use GET.

Generated from the code (api_jobs.php, api_create_story_ai.php, api_social.php, api_tree.php, api_content.php). See also docs/architecture/api-endpoints.md.

Create a full story with AI

POST /api_create_story_ai.php

Tags: Story creation

Resolves "Any" genre/tone/audience to concrete random values, fills a blank premise from a seed, creates a draft story, and queues a story job. Returns immediately (the browser polls the job queue). Note this endpoint uses ok as its result key, not success. Requires login and ai_enabled.

200 OK

Job queued — or a handled error (rate limit / validation) with ok:false.

401 Unauthorized

Not logged in.

403 Forbidden

AI features disabled.

405 Method Not Allowed

POST required.

Job queue actions (create, cancel, retry, apply)

POST /api_jobs.php

Tags: Jobs

Dispatches on action. create submits an image or scene job from the editor (for scene jobs the server augments input_json with story context + the backward path walk). cancel works only while the job is still pending. retry resets a failed job to pending. apply re-applies a completed job's stored result.
